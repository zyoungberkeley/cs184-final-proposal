Procedurally Generated Clouds

Summary:

We would like to explore the rendering of volumetric clouds and gases. In order to facilitate, this project will utilize the GLSL shader language along with rendering techniques like raytracing, bounding volume hierarchies, and other real-time techniques.

Our group consists of William Dai, Callam Ingram, Roman Taylor, and Zachary Young. All of us are CS184 students.

Problem Description:

Rendering solids, mirrors, and refractive materials has been accomplished in previous projects so exploring other materials like volumetrics clouds or gases is a logical next step. This is important to accurately render realistic scenes that have fog mixed with other actors or when actors are flying like in flight simulators. This is challenging due to it requiring real-time rendering combined with procedurally generated data.

To solve this problem of rendering volumetric clouds, we will utilize published papers combined with techniques learned in this course such as bounding volume hierarchies, writing GPU shaders, and other real-time optimization techniques.

Goals and Deliverables:

Inspired by the real time rendering ideas post on Piazza, we decided to further explore the topic of generating realistic clouds in the sky. From further research, our group stumbled upon an awesome video about generating clouds on YouTube. From here we decided to create a real-time cloud simulation. We plan to deliver a working application that generates clouds procedurally with the ability for the user to change the clouds’ appearance in real-time. We will explore multiple avenues for rendering including utilizing code we’ve already written and Shadertoy. Our success metric is to deliver realistic looking clouds rendered in real-time. But to realistically measure our efforts in quality, accuracy, and realism, we will use our gathered resources, combined with various research papers, to compare our results. With our analysis we plan to render volumetric clouds in various ways in order to determine a balance between realism and computational complexity.

Additionally, if time allows, we hope to create an interactive Graphical User Interface to allow free movement through space and add additional details such as varying lighting (sun rise, mid-day, sunset, moonlight etc) and other textures / objects such as mountains below the cloud cover. Some challenges that we might expect to encounter in implementing these additional features include accurately rendering occluded surfaces that might be visible through the clouds, and shading all objects with the proper photorealistic texture. Achieving the result seen at this stock video would be both aesthetically satisfying and be sufficiently different from the tutorial examples we found on Youtube (see fig 2 and resources section for more info) to allow implementing changes of our own. With a fully functional GUI, it should also be possible to tilt and pan the scene, as if the viewer were on an aircraft with varying yaw, roll, and pitch. We hope that our demo will include a GUI with an ability to move through space.

Project Timeline:

Week 1: We explore the existing literature around procedurally generated landscapes, raymarching, and layering landscape elements (i.e. cloud cover over mountains, or over oceans). We decide on the technologies required and set up our work environment.

Week 2: Begin implementation, narrow down goals, have working minimalistic demo. At minimum we need to create a tessellating random cloud texture (modified Worley noise algorithm), adapt it for 3D space, apply shading for every pixel, implement tests for object collision (how much of the other objects should be obscured by the clouds), and integrate it with a handful of environment textures.

Week 3: Add features to add realism, add GUI, complete any moonshots. Implement toggle feature for switching between daylight and moonlight, potentially adding arrow key controls to let the user move the camera in various directions.

Week 4: Refine deliverables, finish building presentation materials. Finalize documentation of our implementation and the key insights gained.
